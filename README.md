# 球王之路

一个本地离线运行的纯文字足球球员生涯模拟游戏。玩家从 16 岁球员开始，在训练、比赛、合同、转会、经纪人建议、媒体发布会和国家队征召之间做选择，逐步走向世界足坛。

项目是纯前端静态网页：不需要后端、不需要数据库、不需要安装依赖。打开 `index.html` 即可游玩；在支持 Service Worker 的安全来源下，也可以安装成 PWA 离线应用。

## 在线/本地运行

### 直接打开

双击打开 `index.html` 即可开始游戏。

这种方式最简单，适合电脑本地试玩。由于浏览器安全限制，直接用 `file://` 打开时不一定能注册 PWA 离线缓存，但游戏本体和本地存档可以正常使用。

### 本地静态服务器

如果想模拟网页部署环境，或者测试 PWA，可以在项目目录运行：

```bash
python -m http.server 4173
```

然后打开：

```text
http://localhost:4173/index.html
```

也可以使用任意静态服务器，例如 VS Code Live Server、Nginx、Caddy、GitHub Pages 或其他静态托管服务。

## 玩法概览

- 创建 16 岁球员：填写姓名，选择国籍、位置和开局身份。
- 两种开局：豪门青训球员或小球队一线队员。
- 每周推进：可以训练、查看下一场比赛、文字播报比赛或跳过直接看赛果。
- 能力成长：训练和比赛会积累成长经验，生涯页会显示距离下次成长还差多少经验和预估训练/比赛次数；训练区会提示疲劳/士气风险，高表现赛季有机会突破原有潜力上限。
- 比赛结算：联赛、欧战和国家队比赛都会结算出场时间、评分、进球、助攻、扑救、疲劳、士气和球队结果；欧战会作为正式下一场比赛进入赛程预告和赛后媒体。
- 积分榜：当前联赛自动生成赛程，模拟其他球队比赛，并显示欧战、升级、降级等资格区；进入欧战后会按新欧战赛制显示联赛阶段积分榜、逐轮生成的淘汰赛路径、进球榜、助攻榜和评分榜，豪门与欧战常客的榜单会优先使用内置真实核心球员池。
- 欧战赛制：欧冠/欧联先踢 8 场联赛阶段，欧协联先踢 6 场联赛阶段；前 8 名直接进入 16 强，第 9-24 名进入淘汰赛附加赛，之后才逐轮生成 16 强、8 强、半决赛和决赛。
- 经纪人系统：经纪人会带来续约、报价、青训挖角、转会兴趣和推荐策略。
- 转会窗：夏窗和冬窗会持续生成正式报价、外租邀请、市场流言、俱乐部买卖人和母队放人判断；同一窗口内每周都可能追加新报价。
- 合同系统：合同年限会随赛季流逝，续约、自由身离队和薪水变化都由表现与关系影响。
- 媒体与舆论：发布会、混采区追问、球迷频道和网络评论会根据出场时间、比赛表现、转会窗与外租/转会策略变化，回答会影响主帅、董事会、球迷、队友、经纪人、士气等关系。
- 国家队：能力、声望、稳定出场和赛季评分会影响国家队征召概率，并参与世界杯、欧洲杯、美洲杯、亚洲杯等大赛荣誉结算。
- 荣誉室：记录联赛冠军、联赛金靴/金球、欧冠金靴/冠军、金球奖、世界杯/欧洲杯等个人和赛事荣誉。
- 手机适配：竖屏下按钮、日志、积分榜和档案信息都可以滚动操作。

## 支持的界面

游戏主界面分为五个视图：

- 生涯：查看时间线、训练并推进每周事件。
- 比赛：查看下一场对手、赛季数据，并选择文字播报或直接赛果；联赛和欧战都会进入正式赛程预告与赛后媒体。
- 积分榜：查看当前联赛排名、进失球、积分和资格区，以及欧战联赛阶段积分榜、淘汰赛路径和球员榜。
- 转会：调整经纪人策略，区分寻求外租和永久转会，并查看、处理正式报价。
- 生涯记录：查看球员资料、能力值、转会记录、俱乐部/国家队生涯、荣誉室、关系与存档导入导出。

## 存档说明

游戏使用浏览器 `localStorage` 本地存档，存档键为：

```text
road-to-king-career-save-v1
```

PWA 离线缓存只负责保存网页文件，不会替代或清空你的生涯档案。

档案页提供：

- 导出存档：把当前存档 JSON 复制到剪贴板。
- 导入存档：粘贴之前导出的 JSON，在新设备或新浏览器继续游玩。
- 重新开档：清除当前浏览器当前来源下的本地存档。

同一个浏览器、同一个网址来源会读取同一份存档。下面这些地址会被浏览器视为不同来源，因此存档互不相通：

```text
file:///.../index.html
http://127.0.0.1:4173/index.html
http://localhost:4173/index.html
http://192.168.x.x:4173/index.html
```

换浏览器、清除网站数据、无痕模式、换网址来源，都可能看不到原来的档。换设备或换来源前，建议先在档案页导出 JSON。

## PWA 与手机离线

项目包含：

- `manifest.webmanifest`
- `sw.js`
- `icons/`

在安全来源下，浏览器可以注册 Service Worker 并安装为 PWA。安全来源通常包括：

- `https://` 静态托管地址
- 本机 `http://localhost`

注意：直接使用电脑局域网 IP，例如 `http://192.168.1.23:4173`，多数手机浏览器不会允许注册离线 Service Worker。这种方式通常可以在线玩，但不保证能安装为离线 PWA。

可选方案：

- 同 Wi-Fi 临时游玩：电脑运行 `python -m http.server 4173 --bind 0.0.0.0`，手机打开电脑 IP。
- 真正手机离线 PWA：把项目放到手机，通过手机本地服务器 App 提供 `http://localhost:端口`，再用手机浏览器添加到主屏幕。
- HTTPS 部署：部署到 GitHub Pages、Cloudflare Pages、Netlify、Vercel 或自己的 HTTPS 静态服务，然后添加到主屏幕。

## 部署

这是一个静态网页项目，可以部署到任何静态托管平台。

以 GitHub Pages 为例：

1. 将仓库推送到 GitHub。
2. 进入仓库 `Settings`。
3. 打开 `Pages`。
4. Source 选择 `Deploy from a branch`。
5. Branch 选择 `main` 和 `/root`。
6. 保存后等待 GitHub Pages 生成访问地址。

部署后建议确认：

- `index.html` 可以正常打开。
- `game.js` 和 `styles.css` 没有 404。
- 浏览器控制台没有 Service Worker 注册错误。
- 手机端可以正常滚动积分榜和日志。

## 开发说明

当前核心逻辑集中在 `game.js` 中，主要包括：

- 球员、俱乐部、联赛和赛程数据。
- 生涯状态创建、保存和读取。
- 训练、成长、比赛模拟、积分榜结算。
- 转会窗、外租、报价、青训合同和续约逻辑。
- 发布会、混采区追问、社交舆论、国家队征召等事件。
- DOM 渲染和按钮事件绑定。

如果修改了 `game.js` 或 `styles.css`，并希望 PWA 用户尽快拿到新版，请同步更新：

- `index.html` 中引用资源的版本参数。
- `sw.js` 中的 `CACHE_NAME`。
- `sw.js` 中缓存列表里的资源路径。

## 隐私

游戏没有后端服务，不会主动上传玩家数据。生涯存档默认只保存在当前浏览器的本地存储中。

如果你把导出的存档 JSON 分享给别人，其中会包含你的球员姓名、生涯状态、日志、俱乐部、比赛和市场信息。
